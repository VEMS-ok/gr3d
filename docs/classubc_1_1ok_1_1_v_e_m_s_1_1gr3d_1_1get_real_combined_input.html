Detailed Description

Combine multiple different classes that implement the PlayerInputs interface. Hence, this can be used as a dropin replacement for anywhere the PlayerInputs is used. Add the different PlayerInputs classes to playerInputsComponents. Note tha each class that implements the PlayerInput would have to be in its own GameObject for this to work. Currently, the Wand and Head would simply return only the values from the first component in playerInputsComponents.
